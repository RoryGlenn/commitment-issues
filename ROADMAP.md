# Roadmap

This document tracks the public direction for `commitment-issues`.

It is intentionally lightweight: no dates, no commitments to shipping order, and no promise that every item will land. The goal is to show what the project is actively improving so users can see where the tool is headed.

## In progress

- Improve onboarding and DX so the first run is easier to understand and safer to preview.
- Expand compatibility guidance for package managers, workspaces, and framework-specific setups.
- Keep the adoption docs current as new commands and workflows land.

## Planned

- Add more recipes for common stacks such as Next.js, Vite, and TypeScript libraries.
- Add CI-provider-specific guidance for disabling Husky in hosted environments.
- Grow the public docs around migration, configuration, and troubleshooting.

## Longer term

- Surface more trust signals in the repository and package metadata.
- Improve discoverability through community writeups, examples, and curated references.
- Keep the advisory-first behavior sharp while making opt-in enforcement clearer.

## How to use this roadmap

- Treat the checklist in [ADOPTION.md](ADOPTION.md) as the source of truth for what has already landed.
- Treat this file as the public view of what the project is still working toward.
- If priorities shift, the roadmap should change with them.
